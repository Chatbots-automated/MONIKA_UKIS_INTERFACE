# Milk Data Testing System - Complete Overview

## What Was Created

A complete system for importing, storing, and managing milk testing data from laboratory websites.

## Database Tables (in Supabase)

### 1. `milk_producers`
Stores farm/producer information:
- Company name, region, collection point
- Producer ID and label (e.g., "naktinis", "rytinis")
- Linked to authenticated users

### 2. `pieno_sudeties_tyrimai` (Composition Tests)
Individual test results for milk composition:
- Fat content (riebalu_kiekis) %
- Protein content (baltymu_kiekis) %
- Lactose content (laktozes_kiekis) %
- pH level
- Urea levels
- Container and protocol numbers
- Test dates

### 3. `pieno_kokybes_tyrimai` (Quality Tests)
Individual test results for milk quality:
- Somatic cell count (thousands/ml)
- Total bacterial count (thousands/ml)
- Non-compliance notes
- Container and protocol numbers
- Test dates

### 4. `milk_scrape_logs`
Audit trail of all import operations:
- When data was scraped
- Date range of imported data
- Number of records imported
- User who performed the import

**Location:** `supabase/functions/import-milk-data/index.ts`

## How n8n Integration Works

```
┌──────────────────┐
│  Playwright      │
│  Scrapes Website │
│  Generates JSON  │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  n8n Workflow    │
│  - Reads JSON    │
│  - POSTs to      │
│    Edge Function │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Supabase        │
│  Edge Function   │
│  - Validates     │
│  - Processes     │
│  - Imports       │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│  Supabase DB     │
│  - milk_producers│
│  - composition   │
│  - quality tests │
│  - logs          │
└──────────────────┘
```

## Deploying the Edge Function

The edge function needs to be deployed to your Supabase project. Since the system doesn't have Supabase CLI, you have two options:

### Option 1: Manual Deployment via Supabase Dashboard
1. Go to your Supabase Dashboard
2. Navigate to Edge Functions
3. Create new function named `import-milk-data`
4. Copy/paste the code from `supabase/functions/import-milk-data/index.ts`
5. Deploy

### Option 2: Use Supabase CLI Locally
```bash
# Install Supabase CLI
npm install -g supabase

# Login
supabase login

# Link to your project
supabase link --project-ref YOUR_PROJECT_REF

# Deploy the function
supabase functions deploy import-milk-data
```

## n8n Configuration Quick Reference

**Endpoint:**
```
POST https://[project-ref].supabase.co/functions/v1/import-milk-data
```

**Headers:**
```json
{
  "Authorization": "Bearer YOUR_USER_TOKEN",
  "Content-Type": "application/json"
}
```

**Body:** Your scraped JSON data (exactly as generated by Playwright)

**Success Response:**
```json
{
  "success": true,
  "imported": {
    "producers": 2,
    "compositionTests": 4,
    "qualityTests": 4
  }
}
```

## Data Format

Your Playwright scraper outputs JSON in this exact format, which the edge function expects:

```json
{
  "scraped_at": "ISO timestamp",
  "url": "source URL",
  "range": {
    "from": "YYYYMMDD",
    "to": "YYYYMMDD"
  },
  "results": {
    "[producer_id]": {
      "gamintojo_id": "...",
      "label": "naktinis/rytinis",
      "meta": { ... },
      "tables": {
        "pieno_sudeties_tyrimai": { rows: [...] },
        "pieno_kokybes_tyrimai": { rows: [...] }
      }
    }
  }
}
```

## Key Features

1. **Automatic Date Parsing**: Converts "25.12.18" to "2025-12-18"
2. **Duplicate Prevention**: Unique constraints prevent duplicate records
3. **Upsert Logic**: Updates existing records, inserts new ones
4. **User Isolation**: RLS ensures users only see their own data
5. **Audit Trail**: Every import is logged with details
6. **Realtime Updates**: Tables support live subscriptions
7. **Error Handling**: Detailed error messages for troubleshooting

## Security

- All tables have Row Level Security (RLS) enabled
- Users can only access their own data
- Edge function validates authentication tokens
- No user can see another user's milk testing data

## Database Performance

- Indexed on producer_id for fast lookups
- Indexed on test dates for date range queries
- Unique constraints on key fields prevent duplicates
- Optimized for both inserts and queries
